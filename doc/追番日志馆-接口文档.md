# 追番日志馆接口文档

## 1. 文档说明

### 1.1 项目名称
追番日志馆

### 1.2 接口风格
- 请求风格：RESTful
- 数据格式：JSON
- 字符编码：UTF-8
- 接口前缀：`/api`

### 1.3 通用响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 1.4 响应字段说明
- `code`：状态码，200 表示成功，其它表示失败
- `message`：提示信息
- `data`：返回数据

### 1.5 常用状态码
- `200`：请求成功
- `400`：请求参数错误
- `401`：未登录或登录失效
- `403`：无权限访问
- `404`：资源不存在
- `500`：服务器内部异常

## 2. 用户模块接口

### 2.1 用户注册
- 请求路径：`POST /api/user/register`
- 接口说明：注册普通用户账号

请求参数：
```json
{
  "username": "test01",
  "password": "123456",
  "nickname": "小雨",
  "email": "test@example.com"
}
```

返回示例：
```json
{
  "code": 200,
  "message": "注册成功",
  "data": null
}
```

### 2.2 用户登录
- 请求路径：`POST /api/user/login`
- 接口说明：用户登录系统

请求参数：
```json
{
  "username": "test01",
  "password": "123456"
}
```

返回示例：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "mock-token",
    "userInfo": {
      "id": 1,
      "username": "test01",
      "nickname": "小雨",
      "role": 0,
      "avatarUrl": ""
    }
  }
}
```

### 2.3 获取当前用户信息
- 请求路径：`GET /api/user/info`
- 接口说明：获取当前登录用户资料

返回示例：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "username": "test01",
    "nickname": "小雨",
    "avatarUrl": "",
    "email": "test@example.com",
    "bio": "热爱补番"
  }
}
```

### 2.4 修改个人信息
- 请求路径：`PUT /api/user/profile`
- 接口说明：修改昵称、头像、邮箱、简介

请求参数：
```json
{
  "nickname": "小雨",
  "avatarUrl": "https://xxx.com/avatar.jpg",
  "email": "test@example.com",
  "bio": "最喜欢治愈番"
}
```

### 2.5 修改密码
- 请求路径：`PUT /api/user/password`
- 接口说明：修改登录密码

请求参数：
```json
{
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

## 3. 番剧模块接口

### 3.1 新增番剧
- 请求路径：`POST /api/anime`
- 接口说明：手动新增番剧信息

请求参数：
```json
{
  "name": "葬送的芙莉莲",
  "originalName": "Sousou no Frieren",
  "coverUrl": "https://xxx.com/frieren.jpg",
  "totalEpisodes": 28,
  "type": "奇幻",
  "sourceType": "漫画",
  "releaseYear": 2023,
  "season": "秋",
  "synopsis": "勇者队伍打倒魔王后的故事"
}
```

### 3.2 分页查询番剧列表
- 请求路径：`GET /api/anime/list`
- 接口说明：分页获取番剧列表，支持按名称和类型筛选

请求参数：
- `pageNum`：页码
- `pageSize`：每页数量
- `name`：番剧名称，可选
- `type`：番剧类型，可选

返回示例：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 1,
    "list": [
      {
        "id": 1,
        "name": "葬送的芙莉莲",
        "coverUrl": "https://xxx.com/frieren.jpg",
        "totalEpisodes": 28,
        "type": "奇幻",
        "releaseYear": 2023
      }
    ]
  }
}
```

### 3.3 获取番剧详情
- 请求路径：`GET /api/anime/{id}`
- 接口说明：根据番剧 ID 获取详细信息

### 3.4 修改番剧信息
- 请求路径：`PUT /api/anime/{id}`
- 接口说明：修改番剧基础信息

### 3.5 删除番剧
- 请求路径：`DELETE /api/anime/{id}`
- 接口说明：删除指定番剧

### 3.6 搜索番剧
- 请求路径：`GET /api/anime/search`
- 接口说明：根据关键词搜索番剧，可为后期接入外部数据源预留

请求参数：
- `keyword`：搜索关键字

## 4. 追番进度模块接口

### 4.1 添加到我的番剧列表
- 请求路径：`POST /api/user-anime`
- 接口说明：将一部番剧加入当前用户的追番列表

请求参数：
```json
{
  "animeId": 1,
  "watchStatus": 0,
  "currentEpisode": 0,
  "remark": "准备开看"
}
```

### 4.2 获取我的番剧列表
- 请求路径：`GET /api/user-anime/list`
- 接口说明：获取当前用户的追番列表

请求参数：
- `watchStatus`：状态筛选，可选
- `pageNum`：页码
- `pageSize`：每页数量
- `keyword`：番剧名称关键字，可选

返回示例：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 2,
    "list": [
      {
        "id": 1,
        "animeId": 1,
        "animeName": "葬送的芙莉莲",
        "coverUrl": "https://xxx.com/frieren.jpg",
        "watchStatus": 1,
        "currentEpisode": 12,
        "totalEpisodes": 28,
        "score": 9.5,
        "isFavorite": 1,
        "lastWatchTime": "2026-04-07 21:00:00"
      }
    ]
  }
}
```

### 4.3 获取追番详情
- 请求路径：`GET /api/user-anime/{id}`
- 接口说明：获取某条用户追番记录详情

### 4.4 更新追番状态和进度
- 请求路径：`PUT /api/user-anime/{id}`
- 接口说明：修改当前集数、状态、评分、备注等信息

请求参数：
```json
{
  "watchStatus": 1,
  "currentEpisode": 13,
  "score": 9.5,
  "isFavorite": 1,
  "startDate": "2026-04-01",
  "finishDate": null,
  "remark": "节奏越来越好了"
}
```

### 4.5 删除追番记录
- 请求路径：`DELETE /api/user-anime/{id}`
- 接口说明：从我的番剧列表中移除某条记录

### 4.6 更新观看进度
- 请求路径：`PUT /api/user-anime/{id}/progress`
- 接口说明：快速更新当前观看集数

请求参数：
```json
{
  "currentEpisode": 14
}
```

### 4.7 获取个人追番统计
- 请求路径：`GET /api/user-anime/statistics`
- 接口说明：获取当前用户追番统计信息

返回示例：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "wantWatchCount": 3,
    "watchingCount": 5,
    "watchedCount": 8,
    "shelvedCount": 1,
    "droppedCount": 1,
    "totalEpisodesWatched": 126
  }
}
```

## 5. 单集日志模块接口

### 5.1 新增单集日志
- 请求路径：`POST /api/episode-log`
- 接口说明：为某部番某一集新增日志

请求参数：
```json
{
  "animeId": 1,
  "userAnimeId": 1,
  "episodeNo": 12,
  "episodeTitle": "真正的勇者",
  "content": "这一集情绪堆得特别好，最后那段太戳人了。",
  "moodTag": "泪目",
  "score": 10.0,
  "isHighlight": 1,
  "watchedAt": "2026-04-07 21:10:00"
}
```

### 5.2 获取某部番剧的日志列表
- 请求路径：`GET /api/episode-log/anime/{animeId}`
- 接口说明：按番剧查询当前用户的日志列表

请求参数：
- `pageNum`：页码
- `pageSize`：每页数量

### 5.3 获取单条日志详情
- 请求路径：`GET /api/episode-log/{id}`
- 接口说明：获取某条日志详情

### 5.4 修改单集日志
- 请求路径：`PUT /api/episode-log/{id}`
- 接口说明：修改日志内容、心情标签、评分等

### 5.5 删除单集日志
- 请求路径：`DELETE /api/episode-log/{id}`
- 接口说明：删除单条日志

### 5.6 获取追番时间线
- 请求路径：`GET /api/episode-log/timeline`
- 接口说明：按时间倒序查询当前用户最近的追番日志

请求参数：
- `pageNum`：页码
- `pageSize`：每页数量

### 5.7 获取神回日志列表
- 请求路径：`GET /api/episode-log/highlight/list`
- 接口说明：获取当前用户标记为神回的日志列表

## 6. 文件上传模块接口

### 6.1 上传图片
- 请求路径：`POST /api/file/upload`
- 接口说明：上传头像、番剧封面等图片资源
- 请求类型：`multipart/form-data`

表单参数：
- `file`：文件
- `bizType`：业务类型，如 `avatar`、`anime_cover`

返回示例：
```json
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "fileName": "frieren.jpg",
    "fileUrl": "https://xxx.com/upload/frieren.jpg"
  }
}
```

## 7. 后台管理接口

### 7.1 后台分页查询用户
- 请求路径：`GET /api/admin/user/list`
- 接口说明：管理员分页查询用户信息

请求参数：
- `pageNum`
- `pageSize`
- `username`
- `status`

### 7.2 修改用户状态
- 请求路径：`PUT /api/admin/user/{id}/status`
- 接口说明：管理员启用或禁用用户

请求参数：
```json
{
  "status": 0
}
```

### 7.3 后台分页查询番剧
- 请求路径：`GET /api/admin/anime/list`
- 接口说明：管理员查看番剧数据

### 7.4 删除日志内容
- 请求路径：`DELETE /api/admin/episode-log/{id}`
- 接口说明：管理员删除违规日志

### 7.5 系统统计接口
- 请求路径：`GET /api/admin/statistics`
- 接口说明：获取用户总数、番剧总数、日志总数等系统统计

返回示例：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userCount": 20,
    "animeCount": 100,
    "userAnimeCount": 230,
    "episodeLogCount": 560
  }
}
```

## 8. 状态与字段约定

### 8.1 追番状态 watchStatus
- `0`：想看
- `1`：在看
- `2`：看完
- `3`：搁置
- `4`：弃番

### 8.2 用户角色 role
- `0`：普通用户
- `1`：管理员

### 8.3 用户状态 status
- `0`：禁用
- `1`：正常

### 8.4 是否收藏 isFavorite
- `0`：否
- `1`：是

### 8.5 是否神回 isHighlight
- `0`：否
- `1`：是

## 9. 开发建议
- 第一版优先完成用户、番剧、追番进度、单集日志四个核心模块
- 接口返回字段尽量与数据库字段一一对应，便于 MyBatis 映射
- 后期可增加 JWT 登录鉴权、Redis 缓存、外部番剧搜索接口
- 前端 Vue 可以按模块拆分为 user、anime、userAnime、episodeLog、admin 五组 API 文件
