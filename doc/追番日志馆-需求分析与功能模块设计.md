# 追番日志馆需求分析与功能模块设计

## 1. 项目概述

### 1.1 项目名称
追番日志馆

### 1.2 项目定位
追番日志馆是一个面向动漫爱好者的个人追番记录网站。用户不仅可以管理番剧观看进度，还可以针对每一集留下自己的观看日志、心情和感想，最终形成带有个人风格的追番记忆档案。

### 1.3 项目目标
- 解决用户忘记看到哪一集的问题
- 为用户提供系统化的追番记录方式
- 支持按集记录感受，沉淀个人追番经历
- 打造区别于传统管理系统的兴趣型产品

## 2. 需求分析

### 2.1 项目背景
很多动漫用户会通过备忘录、聊天记录或笔记软件零散记录自己的追番信息，但这些方式不便于长期管理，也无法系统保存每一集的观后感。常见视频平台通常只能记录播放进度，缺少用户表达和回顾的空间。因此，有必要开发一个兼具进度管理与日志记录功能的追番网站。

### 2.2 目标用户
- 动漫爱好者
- 希望管理追番进度的用户
- 喜欢记录单集感想和追番历程的用户

### 2.3 用户角色

#### 普通用户
- 注册和登录系统
- 添加并管理自己的番剧
- 更新观看进度
- 编写单集日志
- 查看个人追番档案和统计信息

#### 管理员
- 管理用户信息
- 管理番剧基础数据
- 管理日志内容
- 维护系统统计信息

### 2.4 核心需求
- 记录番剧的观看状态
- 记录番剧当前观看进度
- 支持针对每一集填写日志
- 支持按时间回顾追番记录
- 支持后期从外部数据源搜索并添加番剧

## 3. 功能需求

### 3.1 用户端功能

#### 3.1.1 用户注册与登录
- 用户注册账号
- 用户登录和退出
- 修改个人信息
- 修改密码

#### 3.1.2 番剧管理
- 手动添加番剧
- 设置番剧名称、封面、总集数、类型、简介
- 编辑和删除自己添加的番剧
- 后续支持从外部番剧表中搜索并添加

#### 3.1.3 追番进度管理
- 设置番剧状态：想看、在看、看完、搁置、弃番
- 更新当前看到第几集
- 记录开始追番时间和结束时间
- 展示进度条和已看集数
- 按状态分类查看番剧

#### 3.1.4 单集日志功能
- 按番剧和集数创建日志
- 记录文字感想
- 记录心情标签
- 记录单集评分
- 支持编辑和删除日志
- 支持查看某部番剧的所有日志

#### 3.1.5 个人中心
- 查看我的番剧列表
- 查看最近更新的追番记录
- 查看个人追番统计
- 查看追番时间线

#### 3.1.6 搜索与筛选
- 按番剧名称搜索
- 按状态筛选
- 按类型筛选
- 按更新时间排序

### 3.2 管理员端功能
- 用户管理
- 番剧信息管理
- 日志内容管理
- 封面与资源管理
- 基础数据统计

## 4. 非功能需求

### 4.1 界面需求
- 页面风格应具有二次元和兴趣社区特点
- 区别于传统后台管理系统界面
- 支持良好的视觉层次和卡片式展示

### 4.2 性能需求
- 页面常规操作响应及时
- 支持稳定的数据增删改查
- 支持中小规模用户使用

### 4.3 安全需求
- 用户只能操作自己的番剧和日志数据
- 管理员拥有后台管理权限
- 基本的登录校验和权限控制需完整

### 4.4 可扩展性需求
- 后续便于接入外部番剧 API
- 后续便于增加评分、推荐、季度番单等功能

## 6. 业务流程设计

### 6.1 普通用户业务流程
1. 用户注册并登录系统
2. 添加自己想看的番剧
3. 设置番剧状态为想看或在看
4. 更新观看到的集数
5. 为当前集数填写日志
6. 在个人中心查看追番记录和统计信息
7. 看完后修改状态为看完

### 6.2 管理员业务流程
1. 管理员登录后台
2. 查看系统用户、番剧和日志数据
3. 维护番剧基础信息
4. 审核异常日志内容
5. 查看系统统计情况

## 8. 项目亮点
- 不是单纯记录看到第几集，而是记录追番过程中的感受
- 单集日志设计让项目具有内容沉淀价值
- 追番时间线增强了项目的故事性
- 可视化统计可以提升答辩展示效果
- 主题明确，个性鲜明，区别于传统管理系统

## 9. 版本规划

### 9.1 第一阶段必做
- 用户注册登录
- 番剧新增、编辑、删除
- 追番状态管理
- 观看进度记录
- 单集日志记录

### 9.2 第二阶段加分
- 番剧搜索添加
- 追番时间线
- 追番统计图表
- 标签筛选
- 神回标记与收藏

### 9.3 第三阶段扩展
- 接入外部番剧数据接口
- 季度番单展示
- 个性化推荐
- 公开日志或好友互访

## 10. 总结
本项目以动漫追番为主题，将观看进度管理与单集日志记录结合起来，不仅满足了课程设计对业务系统的基本要求，也具备兴趣社区产品的个性化特点。项目既适合使用 SSM 作为后端框架，也适合使用 Vue 构建前端页面，具有良好的可实现性和扩展空间。
